# MTGx — Product Roadmap v2

**Фокус:** максимум полезного функционала. Каждая фича отвечает на вопрос "почему пользователь вернётся завтра?"

---

## Техдолг (сделать первым, 2-3 дня)

Без этого новые фичи будут на гнилом фундаменте.

**rsvp_with_lock.** Текущий код имеет race condition. `FOR UPDATE SKIP LOCKED` в RPC-функции. Без этого waitlist, двухфазная запись и любая логика поверх RSVP — ненадёжны.

**Enum расширение.** Добавить `pending_confirm`, `waitlist` в `rsvp_status` сразу. ALTER TYPE в PostgreSQL необратим — дешевле сделать сейчас.

**supabase gen types.** Заменить ручные типы на автогенерацию. Меньше багов, быстрее разработка.

**Error states.** Пройтись по всем экранам: что видит пользователь при ошибке сети, при "ивент полон", при конфликте данных. Сейчас — ничего. В шумном зале непонятная ошибка = удалённое приложение.

## Волна 1 — Причины возвращаться (1-2 недели)

### «Иду сегодня» — эфемерный сигнал
Большая кнопка на главном экране. Один тап — ты видим всем в городе. TTL 2 часа. Когда 3+ человек нажали — подсказка "Создать встречу?". Это единственная фича которая даёт причину открыть приложение каждый день в 17:00. Заменяет 45-минутный WhatsApp-тред "кто сегодня свободен?".

### XP и уровни
Полноценная система, не заглушка.

Начисление XP:
- Записался going + пришёл (QR check-in): **+20 XP**
- Создал событие которое состоялось: **+30 XP**
- Пригласил нового игрока (реферал): **+50 XP**
- Записался maybe + пришёл: **+10 XP**
- Первый визит в новый клуб: **+15 XP**

Уровни (квадратичная кривая):
| Уровень | XP | Название |
|---|---|---|
| 1 | 0 | Squire |
| 2 | 100 | Apprentice |
| 3 | 400 | Mage |
| 4 | 900 | Archmage |
| 5 | 1600 | Planeswalker |
| 6 | 2500 | Mythic |
| 7 | 3600 | Eternal |

Визуальный бейдж уровня на профиле и в списке участников ивента. Другие игроки видят.

### Достижения (ачивки)
Разблокируются автоматически. Видны на профиле.

| Ачивка | Условие |
|---|---|
| First Tap | Первый RSVP going |
| Regular | 5 ивентов за месяц |
| Organizer | Создал 3 события которые состоялись |
| Multi-format | Играл в 3+ форматов |
| Reliable | 10 going подряд без отмен |
| Explorer | Посетил 3 разных клуба |
| Recruiter | Привёл 3 новых игроков |
| Veteran | 50 ивентов всего |
| Founder | Один из первых 50 пользователей |
| Night Owl | 5 ивентов начавшихся после 20:00 |
| Weekend Warrior | 10 ивентов в выходные |

### QR Check-in
Организатор показывает QR на телефоне/экране. Игрок сканирует = подтверждение присутствия. Это фундамент: XP, reliability, аналитика — всё завязано на реальную явку. Без check-in'а XP за going не начисляется.

### Recap Card
После закрытия ивента — красивая шеринговая карточка: "Friday Draft at Rotemz — 8 players, Pauper." Генерация на клиенте (canvas). Один тап — в WhatsApp. Логотип MTGx = бесплатный маркетинг. Создаёт FOMO у тех кто не пришёл.

### Intent Tags
При создании quick meetup — выбор тега: Casual / Competitive / Deck Testing / Draft / Teaching. Pill chips на карточке. Игрок понимает куда идёт до того как нажмёт going.

### Язык стола
Флаг на событии: RU / EN / HE / Mixed. Иконки флагов на карточке. Для Израиля это критично — русскоязычный игрок, нашедший "русский стол", расскажет всем знакомым.

### Proxy Policy
Переключатель на big event: No Proxies / Partial (N карт) / 100% Proxy Friendly. Иконка на карточке. Убирает сотни одинаковых вопросов.

## Волна 2 — Умная логистика (2-3 недели)

### Двухфазная запись
За 24-6 часов до начала — push "Подтверди участие" с одной кнопкой. Кто подтвердил — цветной аватар в списке. Кто не ответил — серый (maybe). Организатор видит реальную картину. Никаких штрафов — просто уточнение.

### Waitlist
Когда max_players достигнут — кнопка "Встать в очередь". Позиция видна: "Вы #2". При открытии места — push с 30-минутным окном "Место освободилось! Занять?". FIFO, атомарный RPC.

### Авто-отмена
За 3 часа до старта если going < min_players — автоматическая отмена. Push всем: "Не набрался кворум. Перенести?" с кнопкой one-tap reschedule. Не-карательный фрейминг.

### Draft Autosave
Черновик события сохраняется в БД с debounce 2-3 секунды. Индикатор "Сохранено / Синхронизируется". Conflict modal при редактировании с двух устройств. Ивенты создаются в местах со слабым WiFi — потеря черновика недопустима.

### Коммуникация организатор → участники
Push всем записавшимся с готовыми шаблонами: "Начинаем на 30 минут позже", "Нужен ещё 1 игрок — поделись!", "Локация изменилась". One-tap send, 160 символов.

### Reliability Score — расширение
Текущая формула + визуальные статусы:
- **Новичок** (< 3 ивентов) — нейтральный бейдж
- **Надёжный** (> 85% явки) — зелёный бейдж
- **Рискованный** (< 60% явки) — жёлтый бейдж

Видно только организаторам. Никаких блокировок. Никаких штрафов. Просто информация.

## Волна 3 — Клубы и регулярность (3-4 недели)

### Recurring Events
Серия с шаблоном и паттерном: каждый вторник, каждый четверг, раз в 2 недели. iCal RRULE → автогенерация events на 14 дней вперёд. В ленте — одна карточка с бейджем "каждый четверг". Организатор подтверждает одним тапом. Снимает главный pain организаторов — создание одного и того же ивента каждую неделю.

### Календарь клуба
Недельный/месячный вид. Владелец видит: зарезервированные слоты (турниры), спонтанные встречи, свободные окна. Каждое событие с RSVP-счётчиком. Быстрое создание нового ивента в пустом слоте.

### Резервация слотов
Владелец блокирует время под турнир. Слот виден как занятый. Quick meetup в этом слоте невозможен.

### Базовая аналитика для клуба
Без BI, но ценная:
- Уникальные игроки в месяц
- Популярные форматы по дням
- Пиковые дни/часы
- Show rate (записался / пришёл)
- Retention: вернулся после первого визита

### Calendar Integration (.ics)
После RSVP — "Добавить в Google Calendar / Apple Calendar". Стандартный .ics link. Deep link обратно в ивент. Если MTGx ивенты в твоём календаре — приложение sticky.

### Push-шаблоны для организаторов
Готовые сообщения: "Ивент подтверждён!", "Осталось 2 места", "Изменение локации". Организатор выбирает шаблон и отправляет одним тапом.

## Волна 4 — Глубина форматов и discovery (месяц 2-3)

### Commander Brackets
При RSVP на Commander-ивент — опционально указать bracket (1-5). Организатор видит распределение за столом. Решает #1 проблему Commander без rules engine. В будущем — интеграция с Moxfield API для автооценки деклиста.

### Геолокационный поиск
Карта клубов и событий. Радиус +15 км вместо жёсткой фильтрации по городу. Для Гуш-Дана это объединяет Тель-Авив, Рамат-Ган, Петах-Тикву, Реховот. "Что рядом сегодня вечером?" — один тап.

### Store Leaderboard
Публичный рейтинг клубов по частоте ивентов, посещаемости, активности. Дружеская конкуренция между магазинами. Discovery для новых игроков.

### Плейстайл-профиль
Мини-анкета: casual/competitive, скорость игры, общительность. Подсветка совместимости на странице ивента. Не обязательная — заполняется по желанию.

### Сессионные утилиты
Счётчик жизней (Commander с трекингом командирского урона от каждого оппонента), рандомизатор первого игрока, таймер раунда. Каждая минута в MTGx во время игры = retention. Игрок не уходит в другое приложение.

## Волна 5 — Масштаб и монетизация (Q2+)

### Подписка для магазинов (Freemium B2B)
Базовый функционал бесплатно. Pro за подписку:
- Расширенная аналитика
- Приоритетное отображение в поиске
- Брендированные ивенты
- Безлимитные recurring events
- Программа лояльности (Prize Wall) — клуб начисляет баллы привязанные к XP, игроки обменивают на призы

Игроки не платят никогда.

### Лиги / Сезоны
Лига на 2-4 недели с гибкими матчами и standings. Промежуточный формат между турниром и спонтанной встречей. Высокий retention — игроки возвращаются чтобы закрыть сезон.

### WhatsApp/Telegram бот
Команды: /events today, /lfg pauper tomorrow, /my schedule. Подтверждение и отмена прямо из бота. Кросс-постинг ивентов в клубные чаты. Рост без убеждения ставить приложение.

### Иврит + RTL
Полная локализация. RTL layout. Расширение на ивритоязычную аудиторию.

### Платёжная локализация
Bit / Paybox (израильский стандарт). Для турниров с взносом — оплата через клуб, не через платформу.

### Нативное приложение
Capacitor или React Native. Push без ограничений PWA. Иконка в телефоне = постоянное напоминание.

## Что НЕ делать

- Общая лента постов, мемы, новости — есть Discord и WhatsApp
- Трейдинг/обмен картами — другой продукт
- Универсальный чат — убивает фокус
- Субъективные оценки игроков (звёзды/отзывы) — сведение счётов
- Штрафы и блокировки для игроков — отпугивает аудиторию
- Платный функционал для игроков — убивает adoption
- WhatsApp Business API двусторонний бот — 2-6 недель approval Meta, стоимость за сообщение
- GPS-трекинг в реалтайме — Израиль маленький, проблема не "где" а "стоит ли"

## Cold Start

1. Вступить в 15-30 MTG WhatsApp-групп Гуш-Дана
2. Найти 5-10 community hubs — людей которые уже организуют игры
3. Выбрать один магазин — не самый большой, а самый "задолбавшийся" от ручной координации
4. Предложить: "Я лично настрою ваши ивенты. Бесплатно."
5. Прийти физически на первый ивент. Онбордить телефон-в-руке (80% конверсия vs 2% по ссылке)
6. Founder badge — первые 50 пользователей с QR check-in
7. WhatsApp-симбиоз: шаринг text-блоками которые выглядят как написанные человеком
8. Recap Cards после каждого ивента → organic sharing → FOMO

---

## Принципы

**Для игроков — бесплатно, просто, интересно.** Мотивация через XP и ачивки, не через наказания.

**Монетизация — только через магазины.** Freemium B2B.

**Симбиоз с WhatsApp.** Плагин, не замена.

**Каждая фича = причина вернуться.** Если фича не отвечает на вопрос "почему пользователь откроет приложение завтра?" — она не нужна.
